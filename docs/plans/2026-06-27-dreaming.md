# Dreaming (Plan 4) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Add a schedulable `claude-memory dream` maintenance pass — a deterministic exact-duplicate consolidation in the store, plus a headless-Claude semantic reconciliation pass — and a ready-to-paste cron snippet to run it unattended.

**Architecture:** `dream` does two things: (1) a **mechanical** pass — `MemoryStore.dedupeExact()` archives exact-duplicate active memories (no LLM, fully deterministic, testable); (2) a **semantic** pass — shells out to `claude -p "<maintenance prompt>"` so Claude (the brain, already integrated via the user-scoped MCP server) reviews stale/contradictory memories using the `review`/`recall`/`supersede`/`confirm` tools. The command is orchestrated through injected dependencies (config, DB connect, claude runner) so it's testable; a `--print-cron` flag emits a crontab line. This reuses the loop/cron pattern the user started with — Claude on a timer, not a rented OpenAI worker.

**Tech Stack:** Same as prior milestones — Node ≥20, TypeScript (ESM, strict), `pg`, `vitest` + `@testcontainers/postgresql`, `tsup`, `node:child_process` (`execFileSync`).

## Global Constraints

- Node ≥ 20, ESM, TypeScript strict.
- No required LLM key — the semantic pass uses the `claude` CLI (the user's existing Claude Code auth), shelled out via `execFileSync`; the mechanical pass uses no LLM at all.
- `dedupeExact` is namespace-scoped and only touches `status='active'` rows; it keeps one survivor per identical content (most-recently-confirmed, then newest) and archives the rest.
- The `claude` invocation is injectable (an `McpRunner`-style function) so tests never spawn a real process.
- Postgres URL is a secret — never logged.
- The cron snippet uses `$HOME` (not `~`, which crontab does not expand) and redirects to a log file.
- DB-touching tests run against real Postgres via testcontainers.
- Package/bin name: `claude-memory`. MIT.

async dedupeExact(): Promise<number> {
  const { rowCount } = await this.pool.query(
    `with ranked as (
       select id, row_number() over (
         partition by content
         order by last_confirmed_at desc, created_at desc
       ) as rn
       from memories
       where namespace=$1 and status='active'
     )
     update memories m set status='archived'
     from ranked r
     where m.id = r.id and r.rn > 1`,
    [this.namespace]
  );
  return rowCount ?? 0;
}
```

import { execFileSync } from "node:child_process";
import { Pool } from "pg";
import { loadConfig, type Config } from "./config.js";
import { migrate } from "./db/migrate.js";
import { MemoryStore } from "./db/store.js";

export function buildDreamPrompt(): string {
  return [
    "You are running an unattended memory-maintenance pass for claude-memory.",
    "Use the memory MCP tools to keep long-term memory honest:",
    "- Call `review` to list stale memories; for each, `confirm` it if still true, `supersede` it if it changed, or `forget` it if obsolete.",
    "- Use `recall` to find near-duplicate or contradictory memories and reconcile them (supersede the outdated one).",
    "- Keep confidence and provenance accurate.",
    "Be conservative: when unsure, leave the memory unchanged. Do not invent facts.",
  ].join("\n");
}

export function cronLine(schedule = "0 3 * * *"): string {
  return `${schedule} claude-memory dream >> $HOME/.claude-memory/dream.log 2>&1`;
}

export type ClaudeRunner = (prompt: string) => void;

const defaultRunClaude: ClaudeRunner = (prompt) => {
  execFileSync("claude", ["-p", prompt], { stdio: "inherit" });
};

export async function runDreamWith(deps: {
  loadCfg: () => Config;
  connect: (url: string) => Promise<Pool>;
  runClaude: ClaudeRunner;
  log?: (m: string) => void;
}): Promise<{ deduped: number }> {
  const cfg = deps.loadCfg();
  const pool = await deps.connect(cfg.postgresUrl);
  try {
    await migrate(pool);
    const store = new MemoryStore(pool, cfg.namespace);
    const deduped = await store.dedupeExact();
    deps.log?.(`Archived ${deduped} duplicate memories.`);
    deps.runClaude(buildDreamPrompt());
    return { deduped };
  } finally {
    await pool.end();
  }
}

export function runDream(): void {
  runDreamWith({
    loadCfg: () => loadConfig(),
    connect: async (url) => new Pool({ connectionString: url }),
    runClaude: defaultRunClaude,
    log: (m) => console.log(m),
  }).catch((err) => {
    console.error(`dream failed: ${err instanceof Error ? err.message : String(err)}`);
    process.exit(1);
  });
}

export function printCron(schedule?: string): void {
  console.log(cronLine(schedule));
}
```

## Dreaming (background maintenance)

`claude-memory dream` runs a maintenance pass: it archives exact-duplicate memories and then
launches a headless Claude pass (`claude -p`) that reviews stale/contradictory memories and
reconciles them using the memory tools. Schedule it with cron:

```bash
claude-memory dream --print-cron   # prints a crontab line; add it with `crontab -e`
```

The semantic pass requires the `claude` CLI and a valid Claude Code login on the machine that runs
the cron job. Without it, the mechanical dedupe still runs; the semantic pass is skipped/errors and
is logged.
